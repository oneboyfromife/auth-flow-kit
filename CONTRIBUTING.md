## Basic flow

1. Fork the repository on GitHub

2. Clone your fork

```bash
git clone https://github.com/kennethnnabuife/auth-flow-kit.git
cd auth-flow-kit
```

3. Install dependencies

```bash
npm install
# or
yarn install
# or
bun install
```

4. Create a branch from `main`

```bash
git checkout -b docs/readme-update
```

5. Make your changes

6. Commit your changes

```bash
git add .
git commit -m "docs: improve README clarity"
```

7. Push your branch

```bash
git push origin docs/readme-update
```

8. Open a Pull Request to `main`

---

## What you can contribute

- Bug fixes
- Documentation updates
- Small UX / DX improvements
- Example usage (Next.js / React)
- Typos and clarity fixes

---

## Guidelines

- Keep changes small and focused
- Prefer clarity over cleverness
- Avoid unnecessary abstractions
- Open an issue first if unsure

All pull requests are reviewed before merging.
